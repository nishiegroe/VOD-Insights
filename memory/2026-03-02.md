## Integration Points

### With useNativeVideo Hook
All components designed to work seamlessly with the existing hook:

```typescript
const [state, controls] = useNativeVideo({
  filePath: videoPath,
  autoInitialize: true,
});

// Use state in components
<PlaybackControls
  isPlaying={state.isPlaying}
  onPlay={controls.play}
/>
```

### With NativeVideoPlayer
Components ready for multi-component integration:

```tsx
<NativeVideoPlayer src={videoPath}>
  <PlaybackControls {...} />
  <ProgressBar {...} />
  <TimeDisplay {...} />
  {error && <VideoErrorUI {...} />}
</NativeVideoPlayer>
```

## Lessons Learned

1. **Testing library gotchas:**
   - `user.clear()` doesn't work on range inputs (use fireEvent.change)
   - getBoundingClientRect mocking is fragile in tests
   - Screen reader text appears in different places than expected

2. **Component design:**
   - Ref forwarding is essential for parent integration
   - Memoization helps with re-render performance
   - Loading states prevent double-clicks

3. **Accessibility:**
   - ARIA labels needed on every button
   - Keyboard shortcuts must be documented
   - Time display formatting needs flexibility
